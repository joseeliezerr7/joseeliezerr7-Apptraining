1) DATA COLLECTION & SECURITY (top-level questions)
----------------------------------------------------------------

Q: Does your app collect or share any of the required user data types?
A: YES.

Q: Is all of the user data collected by your app encrypted in transit?
A: YES.  (All traffic goes over HTTPS to Supabase + Expo Push.)

Q: Do you provide a way for users to request that their data be deleted?
A: YES — in-app at Profile → Delete account (RPC delete_account()),
   plus a public web URL:
   https://training-app.solucionestechn.com/delete-account

Q: Has your app been independently validated against a global security standard?
A: NO. (No SOC 2 / MASA audit performed.)

Q: Does your app comply with Play Families Policy?
A: NO — app is not directed at children.

2) DATA TYPES COLLECTED
----------------------------------------------------------------

Mark each item below as COLLECTED. For every one of them:
  • Collected:        YES
  • Shared with 3rd:  NO
  • Optional/Required: see notes
  • Ephemeral:        NO (persisted in Supabase)
  • Purpose:          see notes

────────────  Personal info  ────────────

[x] Name
    Required. Stored in profiles.full_name.
    Purpose: Account management.

[x] Email address
    Required for sign-up / login.
    Purpose: Account management, App functionality.

[x] User IDs
    Supabase auth user UUID. Used to scope all per-user rows.
    Purpose: Account management, App functionality.

[x] Phone number
    OPTIONAL. Stored in profiles.phone (nullable).
    Purpose: Account management.

[x] Other info — "Country"
    Required at sign-up. Stored in profiles.country.
    Purpose: Account management, Personalization (locale hint).

────────────  Photos and videos  ────────────

[x] Photos
    OPTIONAL avatar / profile picture uploaded by the user.
    Stored in Supabase storage. Linked to the user UUID.
    Purpose: Account management (profile picture).

────────────  App activity  ────────────

[x] App interactions
    Video playback progress (video_progress.position_seconds /
    duration_seconds) is persisted so the user can resume across
    devices.
    Purpose: App functionality, Personalization (resume position).

[x] Other user-generated content
    – Bookmarks: list of video/manual IDs the user saved.
    – Video notes: free-text notes the user types against a video.
    Purpose: App functionality.

────────────  App info and performance  ────────────

[x] Other app performance data — "Push notification token"
    Expo push token stored in profiles.push_token.
    Purpose: App functionality (delivering "new video published"
    notifications).

────────────  NOT COLLECTED (leave unchecked)  ────────────
- Financial info, Health & fitness, Messages, Contacts,
  Calendar, Files & docs, Web browsing history, Audio files,
  Location (approximate or precise), Device or other IDs (no
  advertising ID, no MAC/IMEI), Crash logs, Diagnostics,
  Performance traces.


----------------------------------------------------------------
3) DATA SHARING
----------------------------------------------------------------

No data is shared with third parties for advertising, analytics, or
any other purpose. All data is processed by:
  • Self-hosted Supabase (Postgres + storage + auth) operated by
    Soluciones Tech on its own infrastructure.
  • Expo Push Service (Apple APNs / Google FCM transport only)
    when sending push notifications. The push payload contains
    only the notification text + video deeplink, no PII.

→ For every data type above, answer "Not shared".
